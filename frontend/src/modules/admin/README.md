# AdminModule (BLOCK 20)

## Назначение
Административный UI для управления доступами через роли:

1. Roles → Modules → Permissions
2. Users → Roles

## Ограничения
- Модуль не вычисляет эффективные права пользователя.
- Модуль не применяет permissions в runtime.
- Изменения доступа вступают в силу после нового логина пользователей.
- Backend остаётся единственным источником истины.
